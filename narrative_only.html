The DotMat is a free-to-use, hosted, mobile responsive visual with built in ability for story telling. Much like the humble pie chart, DotMat communicates part-of-whole relationships. However, unlike the pie chart, DotMat retains the individuality of the entities e.g. people, companies, etc. in each slice. The inspiration for the DotMat came from FiveThirtyEight.com’s visual on gun deaths.

In July 2016, FiveThirtyEight.com used a very simple yet effective visual to talk about gun deaths in America. The visual answers the question - How many people from various subsections of society killed by gun violence? This insight could have been communicated by aggregate charts like Bar Charts, Trellis or Pie Charts. However, use of aggregate visuals would have hidden away the fact that each death is a human life lost. Instead FiveThirtyEight used dots to represent each person killed. This is what makes this visualisation special.



ICFJ.org commissioned this as a reusable visual for the data journalists at IndiaSpend.com. You can use it too.

Tips & tricks

1.

Keep the narrative short, and use it to tell a well-defined, choreographed data-driven story. For instance, a complete summary of Indian MPs, by gender, networth, criminal records etc.
2.

Pick stories where you are showing proportions of countable entities e.g. people, companies, etc.
3.

Use exploratory mode if you want to allow readers to explore raw data and discover insights themselves.
4.

Use data which has absolute values. Using percentage data (i.e plotting 100 points where one point represents one percent) is not the ideal way to use this chart.


Make a DotMat
1
Choose a visual type
  • Narratives & Exploratory
  • Narrative only
  • Exploratory only
2
Create your spreadsheet

Build a new Google Spreadsheet by going to your own Google Drive account. Refer our template. The template has 4 sheets - config, narrative, filters and data. For more details, refer technical documentation.

  • If you intend to publish a narrative-only story, then only include config and narrative sheets. (template)
  • If you intend to publish an exploratory-only story, then only include config, filters and data sheets. (template)
  • If you intend to publish a both story, then include all 4 sheets.

Note:

  • Do not change the sheet names. They should still read - data, config, filters and narrative.
  • Do not change the column headers (first row) in the config, narrative and filters sheet. You can add your own columns in the data sheet.
  • Do not leave any blank rows in your spreadsheet.

Read more on what type of data to enter in the "data" sheet here.

3
Publish the Google Spreadsheet to WEB

Under the file menu, select "Publish to the Web". You'll see 2 select boxes under 'link' tab. The default options selected will be "Entire Document" and "Web Page". Don't make any changes here. In the next window, click the blue "publish" button. When asked, "Are you sure…?" click OK. Copy the URL

4
Embed your visual

Copy the code from below into a Text Editor and replace the Google Sheet URL in the text with your Google Sheet URL. Paste the embed code in your site where you want your visual to appear.
